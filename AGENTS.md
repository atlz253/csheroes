# Инструкции для агентов

## Контекст проекта

`csheroes` - игра на C# в духе `Heroes of Might and Magic`. В репозитории сейчас есть две клиентские оболочки:

- `csheroes` - оригинальный WinForms-клиент на `net6.0-windows`;
- `CSHeroes.Web` - web-клиент на Blazor WebAssembly с рендерингом через Canvas;
- `CSHeroes.Core` - общий слой для web-клиента, который подключает оригинальные исходники игровой модели из `csheroes/src`;
- `CSHeroesTests` - тесты, включая проверки web/core-сессии.

Текущий курс миграции: не переписывать игровую модель с нуля. Web-версия должна использовать оригинальные классы игры как источник правил, состояния, загрузки карт и сохранений.

## Архитектурные правила

1. Оригинальные игровые модели находятся в `csheroes/src`. Не дублируй их в `CSHeroes.Core`.
2. `CSHeroes.Core` подключает оригинальные файлы через linked source в `CSHeroes.Core.csproj`.
3. Новую web-логику добавляй как адаптеры вокруг оригинальной модели:
   - `GameSession` - управление экраном, исследованием карты, сохранением, привалом;
   - `BattleSession` - логика web-боя и AI;
   - `GameSnapshots` - DTO для рендера;
   - `Geometry` - платформонезависимые типы `GridPoint`, `SpriteRect`, `GameConstants`.
4. Canvas-рендер должен строиться только из snapshot-данных Core. Не протаскивай WinForms-типы в Blazor-компоненты.
5. WinForms-клиент должен оставаться рабочим. Не меняй его поведение ради web-клиента без явной необходимости.
6. Не добавляй зависимость WinForms-проекта от `CSHeroes.Core`: это создает конфликты типов, потому что Core тоже компилирует оригинальные исходники.
7. Существующий бинарный формат карт и сохранений нужно сохранять совместимым.

## Web-клиент

Основной экран web-версии находится в `CSHeroes.Web/Pages/Index.razor`.

Canvas interop находится в `CSHeroes.Web/wwwroot/js/canvasRenderer.js`.

Статические ассеты лежат в `CSHeroes.Web/wwwroot/assets`:

- `textures.png` - spritesheet;
- `Menu.png`, `camp.png`, `win.png`, `defeat.png` - фоновые изображения экранов;
- `maps/FirstMap`, `maps/SecondMap`, `maps/ThirdMap` - оригинальные бинарные карты.

Сохранения в web-версии хранятся в `localStorage` как base64-строки.

## Важные игровые нюансы

При исправлении багов проверяй эти случаи отдельно:

- глобальная карта имеет видимую ширину 25 клеток, хотя бинарная карта содержит 26 сериализованных колонок;
- герой на глобальной карте не должен проходить через стены, препятствия и вражеские армии;
- вражеская армия на глобальной карте является препятствием, но клик по ней должен начинать бой, если к ней можно подойти с соседней достижимой клетки;
- после победы в бою миниатюра побежденной армии должна исчезать с глобальной карты;
- melee-юнит в бою должен подходить к цели перед атакой, если цель достижима;
- AI-юниты в бою должны ходить по достижимым клеткам, а не телепортироваться и не застревать без причины;
- экран привала должен показывать фон `camp.png`, а Canvas карты не должен перекрывать этот фон.

## Команды

Сборка solution:

```powershell
dotnet build csheroes.sln
```

Тесты:

```powershell
dotnet test csheroes.sln
```

Запуск web-клиента:

```powershell
dotnet run --project CSHeroes.Web\CSHeroes.Web.csproj
```

Если нужен фиксированный порт:

```powershell
dotnet run --project CSHeroes.Web\CSHeroes.Web.csproj --urls http://localhost:5100
```

## Проверки перед завершением задачи

Для изменений в логике игры запускай:

```powershell
dotnet test csheroes.sln
```

Для изменений в web UI или Canvas дополнительно запускай web-клиент и проверяй в браузере:

- меню открывается;
- каждая карта стартует;
- клик по карте корректно перемещает героя;
- бой начинается по клику на доступного врага;
- юниты ходят и атакуют;
- привал отображает фон и панель слотов;
- сохранение и загрузка работают.

## Стиль изменений

- Делай маленькие, проверяемые исправления.
- Не переписывай модель игры без необходимости.
- Сначала ищи причину в оригинальной модели и адаптерах Core, потом в Canvas/UI.
- Не меняй ассеты и карты механически.
- Не коммить логи локального сервера (`webserver.out.log`, `webserver.err.log`) и временные файлы.
- Если добавляешь новый тест, называй его по проверяемому поведению.
